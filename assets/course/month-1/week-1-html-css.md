# Month 1 Week 1: HTML & CSS 基礎

> **目標**：建立第一個靜態網頁，理解網頁如何構成（結構）與如何變美（樣式）。
> **核心哲學**：就像蓋房子，HTML 是鋼筋水泥（結構），CSS 是裝潢設計（樣式）。

---

## Day 1: HTML 基礎 (The Skeleton)

### 1. 什麼是 HTML？
HTML (HyperText Markup Language) 不是程式語言，而是一種「標記語言」。它告訴瀏覽器：這是標題，那是圖片，這是一段文字。

#### 💡 核心概念：HTML 就像「房子的藍圖」
- **`<html>`**：整棟房子的地基邊界。
- **`<head>`**：房子的機房與水電配置（你看不到，但對房子運作很重要，如 SEO 設定、引入 CSS）。
- **`<body>`**：房子的室內空間（所有你主要看到的東西，如客廳、家具）。

### 2. DOM (Document Object Model) 解析
DOM 是瀏覽器將 HTML 轉換成程式可以理解的「樹狀結構」。

#### 🌳 比喻：族譜樹 (Family Tree)
DOM 就像一個大家庭的族譜：
- **`<html>`** 是老祖宗 (Root)。
- **`<head>`** 和 **`<body>`** 是他的兩個孩子 (Children)。
- **`<h1>`** 和 **`<p>`** 在 `<body>` 裡面，所以他們是 `<body>` 的孩子，彼此是兄弟姊妹 (Siblings)。

> **為什麼這很重要？** 因為之後我們用 JavaScript (像是一個管家) 來根據這張族譜找到特定的家庭成員（元素），並改變他們的狀態（比如把客廳漆成紅色）。

### 3. 常見標籤 (Tags)
- **`<h1>` - `<h6>`**：標題。`<h1>` 最重要（像報紙頭條），`<h6>` 最小。
- **`<p>`**：段落 (Paragraph)。一般的文字敘述。
- **`<div>`**：盒子 (Division)。這是一個沒有特殊意義的容器，就像一個紙箱，用來把東西分組。
- **`<span>`**：標籤。用來標記文字中的一小段（例如把句子中的 *某個詞* 變色）。
- **`<img>`**：圖片。注意它是「自閉合」標籤 (Self-closing)，不需要 `</img>`。
- **`<a>`**：連結 (Anchor)。用來從這一頁跳到那一頁。

## Day 2: CSS 選擇器 (The Stylist)

### 1. 什麼是 CSS？
CSS (Cascading Style Sheets) 負責網頁的「樣式」。如果 HTML 是素顏，CSS 就是化妝品。

### 2. 選擇器 (Selectors) - 如何找到你要化妝的對象？

#### 🎯 比喻：在人群中叫人
- **標籤選擇器 (Tag Selector)**：`p { color: red; }`
    - 比喻：對著人群大喊「所有**人類**都穿上紅衣！」（範圍最大，所有 `<p>` 都會變）。
- **類別選擇器 (Class Selector)**：`.highlight { color: blue; }`
    - 比喻：對著人群喊「所有**戴別針的人**穿上藍衣！」（最常用，透過 `class="highlight"` 指定）。 
- **ID 選擇器 (ID Selector)**：`#navbar { background: black; }`
    - 比喻：對著人群喊「**身分證號碼是 A123 的那位**穿上黑衣！」（獨一無二，一個頁面只能有一個）。

## Day 3: 盒模型 (The Box Model)

### 1. 萬物皆盒子
在 CSS 的世界裡，**每一個** HTML 元素（不論是一張圖、一段字）本質上都是一個長方形的盒子。

### 📦 比喻：精心包裝的禮物盒
試想你在包裝一個易碎的禮物：
1.  **Content (內容)**：真正的禮物本身（例如一隻手錶）。
    - 這是 `width` 和 `height` 設定的大小。
2.  **Padding (內距)**：禮物周圍塞的防撞氣泡紙。
    - 它是透明的，位於內容和盒子邊緣之間。背景顏色會延伸到這裡。
3.  **Border (邊框)**：裝禮物的紙箱殼。
    - 這是你看得見的邊線。
4.  **Margin (外距)**：這個禮物盒和其他禮物盒之間的距離。
    - 這是盒子外部的透明空間，用來推開別人。

> **重要觀念：`box-sizing: border-box;`**
> 預設情況下，`width` 只算禮物的寬度。如果你加了氣泡紙 (Padding) 和紙箱 (Border)，整個體積會變大。
> 設定 `box-sizing: border-box;` 就像告訴工人：「我不管你氣泡紙塞多少，這顆包裹總寬度就是不能超過 30 公分！」這會讓排版容易得多。

## Day 4: Flexbox 佈局 (The Layout Manager)

### 1. 為什麼需要 Flexbox？
在 Flexbox 出現前，要把東西並排顯示（比如導航選單）非常痛苦。Flexbox 就像是一個「智慧型排版管家」。

### ✨ 比喻：彈性極佳的收納盒
當你在一個父容器 (Container) 設定 `display: flex;` 時，你就像啟動了一個魔法收納盒：
- **主軸 (Main Axis)**：預設是水平的（橫向排列）。所有子元素會乖乖排成一列。
- **交錯軸 (Cross Axis)**：預設是垂直的。

### 2. 兩個最重要的指令
- **`justify-content` (主軸對齊)**：或是說「如何分配剩下的空間」。
    - `center`：全部置中。
    - `space-between`：第一個貼左邊，最後一個貼右邊，中間平均分配（最常用於導航列）。
- **`align-items` (交錯軸對齊)**：通常是垂直對齊。
    - `center`：垂直置中（終於不用算像素了！）。

## Day 5: 響應式設計 (Responsive Design)

### 1. 變色龍網頁 (The Chameleon)
現在的網頁必須在手機、平板、電腦上都好看。響應式設計就是讓網頁像變色龍一樣，根據環境（螢幕寬度）改變外觀。

### 2. 核心技術：Media Queries
這是 CSS 的條件判斷：
```css
/* 當螢幕寬度小於 600px 時 (手機版) */
@media (max-width: 600px) {
  .menu {
    flex-direction: column; /* 原本橫向的選單變垂直堆疊 */
    background-color: #333;
  }
}
```

### 💧 比喻：水與容器
響應式設計的內容應該像「水」一樣。
- 寫死 `width: 800px` 就像一塊冰塊，放到小杯子裡裝不下。
- 使用 `width: 100%` 或 `max-width` 就像水，會自動適應杯子的形狀。
