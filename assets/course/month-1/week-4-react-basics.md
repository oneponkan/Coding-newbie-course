## Day 1: 元件 (Components) & JSX

### 1. 什麼是 React？
React 是一個用來「構建使用者介面」的 JavaScript 函式庫。它的核心思想是 **Component-Based (元件化)**。

#### 🧱 比喻：樂高積木 (LEGO Bricks)
- **傳統網頁**：像是在捏黏土。你想做一個城堡，你從頭捏到尾，如果要改城門，可能整個牆都要重捏。
- **React**：像是在組樂高。
    - 你先做出一個小積木叫 `<Button />`。
    - 再做出一個積木叫 `<Navbar />`。
    - 最後像拼圖一樣，把這些積木組合成 `<App />` (你的城堡)。
- **好處**：如果有天你想把所有按鈕從藍色改成紅色，你只要改 `<Button />` 那個積木的設計圖，全世界的按鈕都會變紅色。

### 2. JSX (JavaScript XML)
你看到的 `<Button />` 其實不是 HTML，它是寫在 JS 裡的 HTML 語法糖。
- 它讓你在邏輯代碼 (JS) 裡面直接寫 UI 結構 (HTML)，合二為一。

## Day 2: Props (屬性)

### 1. 傳遞資料給元件

#### 📝 比喻：客製化訂單 / 藍圖指令
如果不傳參數，每個 `<Button />` 長得都一樣。Props 就像是你給積木工廠的**訂單指令**。
- 父元件 (Parent) 對子元件 (Child) 說：「給我一個按鈕，文字是"登入"，顏色是"藍色"。」
- 子元件收到 `props` (訂單)，依照指示渲染出對應的樣子。

#### ⚠️ 鐵律
- **Props 是唯讀的 (Read-Only)**：
    - 就像客人點了「巧克力蛋糕」，廚師(子元件)不能擅自把它改成「草莓蛋糕」。
    - 資料是單向流動的 (由上往下流)。

## Day 3: State (狀態)

### 1. 元件的記憶

#### 🧠 比喻：短期記憶 / 廚房冰箱
- **Props** 是別人給你的指令（外來的）。
- **State** 是你自己當下的狀況（內在的）。
- 比如一個 `<Counter />` 元件：
    - 它現在數到幾了？ (`count = 5`)
    - 這是它自己的私事，不用問爸爸，也不受別人控制。
    - 當這個 `state` 改變時（5 變成 6），React 會自動幫你更新畫面（Re-render）。

#### 💡 Props vs State 口訣
- **Props**: 爸爸給的零用錢 (你不能決定給多少，只能花)。
- **State**: 你錢包裡剩多少錢 (隨你花費變動，你自己管)。

## Day 4: Hooks (魔法背包)

### 1. 賦予函數元件超能力
以前的函數元件很笨，只會顯示畫面。Hooks 就像給它穿上了裝備。

- **`useState`**: 讓元件擁有「記憶」的能力。
    - 就像給了它一個筆記本，可以記住 "count" 是多少。
- **`useEffect`**: 讓元件擁有「感知」的能力。
    - 就像設了一個鬧鐘或感應器：「當 [某變數] 改變時，就去做 [某件事]。」
    - 常可以用來：一開機就去抓資料 (Fetch API)、監聽滑鼠移動。
