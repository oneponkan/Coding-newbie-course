# Month 1 Week 3: TypeScript (The Safety Net)

> **目標**：從「會動」進階到「不會隨便壞掉」。
> **核心哲學**：預防勝於治療。TypeScript 是 JavaScript 的保全系統。

---

## Day 1: 為什麼要用 TypeScript？

### 1. JavaScript 的自由與危險
使用純 JavaScript 就像在**沒有速限也沒有車道線的空地**上開車。
- **優點**：自由自在，想怎麼開就怎麼開。
- **缺點**：容易相撞，容易迷路。如果有人把「汽車」當成「飛機」開，系統不會阻止，直到摔下來。

### 2. TypeScript 是什麼？
TypeScript = JavaScript + **Type System (型別系統)**。

#### 🛡️ 比喻：工地安全檢查員 (Safety Inspector)
TS 就像一個嚴格的檢查員，在你**蓋房子之前 (Compile Time)** 就先檢查藍圖。
- JS：房子蓋好了，住進去才發現樑柱是紙糊的 -> **倒塌 (Runtime Error)**。
- TS：還沒蓋之前，檢查員就說：「嘿！這裡規定要用鋼筋，你怎麼寫成紙板？」 -> **修正 (Compile Error)**。

## Day 2: 型別註解 (Type Annotations)

### 1. 給變數貼標籤

#### 🏷️ 比喻：形狀分類箱
兒童玩具箱上有些洞，圓形的洞只能塞圓球，方形的洞只能塞方塊。
- `let score: number = 100;`
    - 這個變數宣告它是「數字洞」，你如果硬要塞字串 `"hello"` 進去，TS 會亮紅燈擋住你。

### 2. 常見標籤
- `string`: 文字
- `number`: 數字
- `boolean`: 是非
- `any`: **萬用垃圾桶** (盡量少用！這等於把檢查員趕出去，一切回到 JS 的無政府狀態)。

## Day 3: 介面 (Interface)

### 1. 定義物件的「合約」

#### 📜 比喻：租屋合約 (The Contract)
當你定義一個 `Interface`，你就是在寫一份合約。
```typescript
interface User {
  name: string;
  age: number;
  isMember: boolean;
}
```
這份合約規定：「要成為一個 `User`，你**必須**有一個名字(字串)、一個年齡(數字)和一個會員狀態(布林)。」

- 如果你創造一個 User 卻**少了**年齡 -> **違約 (Error)**。
- 如果你把年齡**寫成**文字 -> **違約 (Error)**。
- 這保證了你的程式碼中，所有叫 `User` 的東西，長得都一模一樣，不會有驚喜。

## Day 4: 泛型 (Generics) 入門

### 1. 靈活但安全的盒子

#### 📦 比喻：透明包裝機
想像你有一個機器，它的功能是「把東西包起來」。
- 如果沒有泛型，你可能要買三台機器：「包蘋果機」、「包香蕉機」、「包西瓜機」。
- 有了泛型 `<T>`，這台機器變成「通用包裝機」。
    - 你投入蘋果 (`<Apple>`)，它就吐出包好的蘋果。
    - 你投入香蕉 (`<Banana>`)，它就吐出包好的香蕉。
- 它既通用，又知道裡面裝的是什麼（不會把你投入的蘋果變成香蕉）。
